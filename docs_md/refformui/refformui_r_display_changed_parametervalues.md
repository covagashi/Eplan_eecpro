# Display of modified parameter values in radio buttons and drop-down list boxes

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Display of modified parameter values in radio buttons and drop-down list boxes If the same parameter value is displayed in multiple Form-UIs, an internal validation prevents, that non-editable drop-down list boxes display invalid parameter values. Example of radio buttons and drop-down list boxes: In two drop-down list boxes and in a field of radio buttons the values of the parameter CardinalPoints should be selectable in parallel (see the following figure). In the left drop-down list box the values are editable, but not in the one on the right. Prior to the modification the parameter CardinalPoints has the value North. So, in both drop-down list boxes (1. and 2.) North is displayed, resp. the radio button (3.) North is marked. **After the modification** of the parameter from North to Northern in the left drop-down list box (4.), the right drop-down list box (5.) does not display any value, resp. non of the radio buttons (6.) is marked.
