## Log file

During the processing of a Job definition, logging is carried out in the console and a log file.

If the Job cannot be processed successfully, a sub-folder with the name of the Job is created below the Folder for Failed Jobs (see User preferences for the Job Server). The log file is then written to this sub-folder.

If the Job is successful, a sub-folder with the name of the Job is created below the Folder for Output Data preferences for the Job Server. The log file is then written to this sub-folder.

If an e-mail is sent in the event of success or an error, it will contain a link with the address of the subfolder that contains the log file in order to allow for quick access to it.
