# Mechatronic

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Mechatronic The mechatronic model is developed according to the maximum principle, so that non-desired components have to be switched off with an option (see also [Generation of variants via disabler](tutmechatronic_h_create_variants_per_disabler.htm)). The following disciplines are based on this mechatronic model:
     * Text
     * Word
     * Eplan Electric P8
     * Eplan Pro Panel
     * SIMATIC STEP 7
     * Project generation per import
     * Job Server
The disciplines are set up in such a way that their components are first built-in fixed into the components of the mechatronic. For experienced users the model in the connection point is rebuilt so that the discipline components can be relocated into their own libraries. Only on request are the discipline components inserted on the extension points of the mechatronic components. The following image schematically shows the conversion to extension points, using the example of the Feeder station. After the mechatronic model been created, the result can be compared with the sample solution (see [Comparison with sample solution](tutorial_h_compare_with_example_solution.htm)).
