# Comparison with sample solution

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Comparison with sample solution In order to compare the created model with the sample solution a preconfigured module is imported as the EOX file: The sample solutions are available under <EEC installation folder>\install\Tutorial\. Discipline | Model of sample solution  
---|---  
Mechatronic | Mechatronic.eox  
Text | ListOfComponents.eox  
ECAD | P8.eox  
ProPanel | ProPanel.eox  
PLC | Step7_Scope1.eox  
Step7_Scope2.eox  
Step7_Scope3.eox  
Step7_Scope4.eox  
Office | Word.eox  
GRAPH2D | Graph2D.eox  
The section [Importing libraries/projects as an EOX file](eecbase_k_eox_exchange_library_import.htm) describes how an EOX file is imported. As an option for the import select Add. The following symbols can be displayed in the synchronization view: Symbol | Meaning  
---|---  
| The incoming changes shown with a plus sign (+) indicate that a component is missing in your solution.  
| The incoming changes shown with a minus sign (-) indicate that there is one more component in your solution than in the sample solution.  
| The incoming changes shown with the red double arrow indicate that action is required on the part of the user.  
Incoming changes generally indicate that your solution differs from the sample solution. Double-click on the incoming change to view further information on the component.
