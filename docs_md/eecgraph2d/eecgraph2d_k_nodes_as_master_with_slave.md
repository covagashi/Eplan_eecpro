## Nodes as master with slave

If a direct dependency exists between components in the mechatronic structure such that the components should always be displayed together, they can be represented as master and slave nodes. If a master node is inserted into the diagram, its slave node is always inserted with it, nor can the latter be deleted without also deleting the former.

The slave node is arranged at a predetermined position on the border of the master node (see [Orientation of master and slave](eecgraph2d_k_nodes_as_master_with_slave_orientation.htm)). This position can be changed later on.

If master is moved, the slave node remains in the same position relative to the master. If the master node is rotated, the orientation of the slave node relative to the master node may change (see [Rotation of master and slave](eecgraph2d_k_nodes_as_master_with_slave_rotation.htm))
