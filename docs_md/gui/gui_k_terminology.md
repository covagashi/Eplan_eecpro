# Terminology

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Terminology All modeled contexts and stored data are mapped in the information model (in short: model) and stored in a database management system or an EOX file. The model is organized by means of assemblies. Assemblies are organizational units. Assemblies can be projects and libraries. To a project, only objects from imported libraries are visible. The sum of libraries necessary to create a project is referred to as the modular system. The objective is to cover the project requirements in as flexible a manner as possible. This is given by the modular system organization. The systems to be integrated, for example, Eplan Electric P8 or Simatic S7, require system libraries. These are supplied with EEC and cannot be modified. They contain the necessary basic components.
