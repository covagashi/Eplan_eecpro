# Debugging against installed EEC

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Debugging against installed EEC You can debug the execution of a self-created native Java method directly against an installed EEC. To this purpose an installed EEC is set as the target for the execution. The execution of the newly created method can be interrupted at any break points. The development environment displays an image of the current memory data for each break point. Subsequent execution of the program is controlled remotely from the development environment. You can have the code executed either step-by-step or as a whole.
