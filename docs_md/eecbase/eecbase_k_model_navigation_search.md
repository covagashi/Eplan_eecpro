## Navigation and search in the model

To navigate within an assembly, one clicks on the object while keeping the [Alt] key pressed. Similar to hyperlinks, the option for navigating can be recognized by the fact that the mouse pointer changes from an arrow to a hand symbol (see the following figure).

The following navigation options exist:

     * From project component to library component.
     * From parameter used to parameter definition.
     * From object in formula result to object definition.

If the formula result is a collection of objects, editors will be opened for all objects.

The following figure shows an example of navigation from a project component to a library component.

EEC comes with a search function. The search comprises the following search functions:

     * [Text search](eecbase_k_model_navigation_search_fulltextsearch.htm)
     * [Search for failed formula results](eecbase_k_model_navigation_search_errorformulasearch.htm)
     * [Formula search](eecbase_k_model_navigation_search_formulasearch.htm)
     * [Disabler Search Page](eecbase_k_model_navigation_search_disablersearch.htm)
     * File search
     * [Unused objects search](eecbase_k_model_navigation_search_unusedobjectssearch.htm)

The text search can take place globally via the  button in the toolbar or via the menu Search > Text search.

The text search can also be started using the key combination [Ctrl] \+ [H].

Alternatively to the global search, a targeted search in selected assemblies or objects is also possible via the popup menu in the Explorer view.

Depending on whether a search is started globally or based on a selection, the search takes place across the entire Workspace or is limited to Selected objects). In addition the search can be limited to parameters, interfaces, mechatronic components, discipline components or a combination of these.
