## Creating project documents

Project documents are generated by means of system-specific generators (see [Generate structure](eecbase_k_linking_dcs_generate_structure.htm)). During the generation process, the discipline-specific components are assembled according to their configuration .

Thus, the components list is created and opened:

For created data, a folder with the name of the project is created in the workspace and, below, a folder with the name of the discipline.

The syntax is as follows:
    
        <Installation folder>\workspace\<Project name>\<Discipline name>\<created data>

For the Word tutorial, the created file is stored as follows:
    
        c:\Users\Public\EPLAN\EngineeringConfiguration\2_4_20141203_2019\workspace\Feeder\Word\ListOfComponents.docx

### Note

It is recommended that you remove files that are no longer required or obsolete versions from the target directory before creating project documents again.

While creating the file with Web EEC it must be ensured, that the created files of different users are stored in accordingly named folders. Therefore during the generation process via Web EEC the folder structure is extended by a further folder, which is given the name of the session ID. The session ID is a sequence of numbers and letters based on the ID of the session on the web server and of an internal EEC session, e.g. 74647372_nf78sdf9s.

The syntax is as follows:
    
        <Installation folder>\workspace\<Project name>\<Session ID>\<Discipline name>\<created Daten>

For the Word tutorial, the created file would be stored as follows (example):
    
        ..\Users\Public\EPLAN\EngineeringConfiguration\2_4_20141203_2019\workspace\Feeder\74647372_nf78sdf9s\Word\ListOfComponents.docx
