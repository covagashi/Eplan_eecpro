## PX update

PX files always need to be updated when projects are to be created by importing archived PX files whose modular systems have meanwhile been changed.

The necessary changes in a PX file can generally be done manually. Following the scheme above, the library component C in the archived project would have to be renamed to B. Then, the import of the PX file works without any conflict. With a large number of PX files and / or old projects, however, the effort of manually updating them can be too much.
