## Manual strategies

Apart from the system strategy, the modular system developer itself offers also a manual strategy for solving the conflict. In the following example, it is not the value (Gripper to grip) from the EOX file that is accepted for merging; instead, the modular system developer first highlights the option Merge value manually, and then enters in the Internal model column a new value (Fast and small gripper. By saving the editor ([Ctrl] \+ [S]), the manually entered value is adopted.

Particularly relational conflicts cannot be solved by a system strategy, but require a manual strategy.

In this case (described in Scenario 4), the option is offered to navigate to the component in the internal model (Navigate to component in internal model) in order to solve the conflict there manually. For example, the relation with the deleted Component can be removed in the internal model.
