# Task: Creating a configuration diagram for office infrastructure

This functionality is only available for certain module packages. [Info / Copyright](license.htm) You are here:  Task: Creating a configuration diagram for office infrastructure A configuration diagram is to be created for the configuration of an office infrastructure: To achieve the objective of this task, this chapter first describes how the Tutorial project, with the architecture and modular library, should be imported by means of an EOX file. Then, the diagram configuration will be created and tested in steps. The configuration starts with the node for building components and room components (devices and persons). In the latter case, the master-slave relationship of nodes is studied. The nodes can first be dragged from the project tree into the diagram. For inputting parameter data, calls are created to pre-finished UIs. Next the definition is expanded with edges (connections), with which persons are connected with the devices, as also, the devices are connected with one another. Thereafter, the palette is extended, so that the nodes can then also be instanced and connected when working from the palette. Finally, the configuration is expanded in such a way that the components of the diagram can be moved at will in the framework of a permitted floor configuration.
